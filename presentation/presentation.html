Intelligent fraud detection using Machine Learning & MLOps

Speaker: Amar and Amir

Speaker Notes: Welcome the audience. Introduce yourself. State the objective: "Today, we are going to explore a modern, scalable solution to one of the biggest cybersecurity threats today: Phishing."

The Problem: The Rise of Phishing

The Threat Landscape

Phishing attacks account for over 80% of reported security incidents.

Attackers use sophisticated social engineering to steal credentials, financial data, and sensitive personal information.

Current Limitations

Traditional systems rely on static blacklists and rigid rules. They cannot adapt to new, cleverly disguised phrasing used by modern attackers.

Speaker Notes: Emphasize that traditional firewalls and spam filters are failing because attackers change their domain names and email phrasing daily. We need a system that understands the *context* and *intent* of a message, not just its sender.

The Solution: Our AI Pipeline

We built a full-stack Machine Learning application that analyzes text in real-time, assigning a risk score based on learned patterns and psychological triggers.

  • Machine Learning Models: Trained on over 10,000 legitimate and phishing emails using TF-IDF vectorization and advanced classifiers.
  • Heuristic Red Flags: Detects specific psychological triggers (Urgency, Money Requests, Deception).
  • Modern Stack: Accessible via a beautiful Streamlit UI and powered by a highly concurrent FastAPI backend.
Speaker Notes: Explain that our solution isn't just a model in a Jupyter notebook. It's a fully productized application. It combines the statistical power of AI with explicit rule-based "Red Flags" so users get an explanation they can actually understand.

System Architecture

Speaker Notes: Walk through the flow. The user types a message in the Streamlit UI. It sends a JSON payload to FastAPI. FastAPI uses NLTK to clean the text, converts it to vectors, runs it through the loaded Pickle model, calculates a risk score, and sends the verdict back.

Data & MLOps Pipeline

To ensure our models are reproducible and trackable, we implemented enterprise-grade MLOps.

Data Version Control (DVC)

We use DVC to track data transformations, training, and evaluation pipelines. If data changes, we just run dvc repro to automatically rebuild the winning model.

MLflow & DagsHub

Every training run logs hyperparameters, F1-scores, and accuracy to DagsHub. The evaluation script automatically selects and exports the best performing model to production.

Speaker Notes: This is a major technical highlight. Mention that without DVC/MLflow, ML projects turn into a mess of "model_final_v2_real.pkl". We automated the pipeline so the backend always serves the mathematically proven "best" model.

Explainable AI: Red Flags

A simple "Phishing" label isn't enough. Users need to know why.

Our backend calculates a Risk Score (0-100) and pairs it with keyword-based contextual analysis:

  • 🚨 Urgency: "Act now", "Expires immediately"
  • 🚨 Suspicious Links: "Click here to verify"
  • 🚨 Financial Deception: "Wire transfer", "Account suspended"

This combined approach provides a human-readable explanation alongside the mathematical confidence score.

Speaker Notes: Explainability is key for user trust. If the AI says a message is 92% risky, the Red Flags engine provides the concrete evidence (e.g., "This message contains urgent language and requests financial details").

Production Deployment (AWS & Docker)

We designed the system to be highly portable and scalable.

Containerization

The entire stack is packaged using Docker & Docker Compose. We maintain separate Dev (hot-reloading) and Prod (optimized) configurations.

AWS EC2 Infrastructure

Currently deployed on an AWS EC2 instance. Traffic is routed through an Nginx Reverse Proxy on port 80 for robust, public internet access.

Speaker Notes: Discuss the deployment strategy. We containerized the frontend and backend. We use `docker-compose.prod.yml` to launch it on a free-tier AWS EC2 instance. It takes less than 3 seconds to spin up the entire application using our pre-trained static models.
